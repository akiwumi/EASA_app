# SQL + Supabase Instructions

> Last updated: 2026-05-01
> Audience: novice
> Goal: set up Supabase for the upgraded flight-school product

## 1. Read This First

This file tells you:

- what to click manually in Supabase
- what SQL to paste into the SQL Editor
- what order to do things in

Important:

- do not run random SQL out of order
- run one section at a time
- confirm each section succeeds before moving on
- if a query fails, stop and fix that before continuing

## 2. Before You Start

You need:

- a Supabase project
- access to the SQL Editor
- access to Authentication settings
- access to Storage
- your `.env.local` file in this project

## 3. Recommended Setup Order

Do the setup in this order:

1. Create the Supabase project
2. Set auth URLs
3. Run base schema SQL
4. Run existing app feature SQL if not already present
5. Run the new flight-school upgrade SQL
6. Create storage buckets
7. Add seed data
8. Add environment variables to the app
9. Test login and basic pages

## 4. Manual Supabase Click Steps

## Step A: Create the project

In Supabase:

- create a new project
- choose a project name
- save the database password somewhere safe

## Step B: Get your keys

In Supabase:

- go to `Project Settings`
- go to `API`
- copy:
  - project URL
  - anon key
  - service role key

Put them in your local `.env.local`.

Example:

```env
NEXT_PUBLIC_SUPABASE_URL=YOUR_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=YOUR_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY=YOUR_SERVICE_ROLE_KEY
```

## Step C: Set auth URLs

In Supabase:

- go to `Authentication`
- go to `URL Configuration`
- set:
  - Site URL to your local app URL, for example `http://localhost:3000`
  - Redirect URLs to include `http://localhost:3000/**`

If you deploy later, add your production domain too.

## Step D: Storage buckets

Create these buckets if they do not already exist:

- `flightbooks`
- `snapshots`
- `exports`
- `source-files`

If a bucket already exists, leave it.

## 5. SQL Execution Strategy

If your database is brand new, run:

1. Base schema block
2. Flight-school upgrade block
3. Dashboard views block
4. Seed block

If your app already has most tables, you can still run these because the SQL uses a lot of `if not exists` protection. Still, read the result messages carefully.

## 8. Dashboard Views SQL

```sql
create or replace view v_training_assignment_status as
select
  da.organization_id,
  count(*) filter (where da.status = 'assigned') as assignments_open,
  count(*) filter (where a.status = 'acknowledged') as assignments_acknowledged,
  count(*) filter (where a.status = 'pending') as assignments_pending_ack
from document_assignments da
left join acknowledgements a
  on a.assignment_id = da.id
group by da.organization_id;

create or replace view v_training_signoff_status as
select
  organization_id,
  count(*) filter (where status = 'pending') as signoffs_pending,
  count(*) filter (where status = 'completed') as signoffs_completed
from training_signoffs
group by organization_id;

create or replace view v_programme_overview as
select
  p.organization_id,
  p.id as programme_id,
  p.name,
  count(distinct ph.id) as phase_count,
  count(distinct l.id) as lesson_count
from training_programmes p
left join training_phases ph on ph.programme_id = p.id
left join training_lessons l on l.programme_id = p.id
group by p.organization_id, p.id, p.name;
```

### Step C

Run this SQL after replacing the placeholders:

```sql
insert into org_users (organization_id, user_id, role)
select
  o.id,
  'YOUR_AUTH_USER_ID'::uuid,
  'admin'::app_role

on conflict (organization_id, user_id) do update
```

## 14. Common Problems

### Problem: `permission denied`

Cause:

- RLS policy is blocking the user

Fix:

- confirm your user exists in `org_users`
- confirm the user has the right role

### Problem: page loads but no data

Cause:

- organization exists, but the user is not linked to it

Fix:

- check the `org_users` table

### Problem: SQL says relation already exists

Cause:

- table was already created earlier

Fix:

- if the statement uses `if not exists`, this is usually safe
- if not, stop and inspect before continuing

### Problem: role type cast fails

Cause:

- old `org_users.role` values do not match the new enum

Fix:

- update old text values first, then cast

Example:

```sql
update org_users
set role = 'viewer'
where role is null;
```
