# EASA App: Novice Deployment Guide for Vercel + Supabase

> Created: 2026-05-02
> Audience: beginner / novice
> Goal: get the fully built app working in production on Vercel and Supabase

This guide tells you exactly what to do next.

It covers:

- which environment variables you need
- which SQL files you must run in Supabase
- which Supabase Edge Functions you must deploy
- what to configure in Vercel
- what to click in the app after deployment
- how to verify that everything is really working

If you follow this guide in order, you should end up with a working production app.

---

## 1. What Is Already Done

The codebase is already built through all 9 roadmap phases.

The big remaining tasks are:

1. Create or confirm your Supabase project

## 2. Before You Start

You should have:

- a Supabase account
- a Vercel account
- this repo on your machine
- this repo in GitHub or another git remote that Vercel can import

You should also know:

- your Supabase project URL
- your Supabase anon key
- your Supabase service role key

Important:

- `NEXT_PUBLIC_SUPABASE_ANON_KEY` is safe for the browser
- `SUPABASE_SERVICE_ROLE_KEY` is **secret** and must only be stored as a server-side environment variable
- never paste secret values into public files in the repo

## 5. Supabase Project Setup

## 5.1 Create the project

In Supabase:

1. Create a new project
2. Wait for the database to finish provisioning
3. Copy these values from **Project Settings**:
   - `Project URL`
   - `anon public key`
   - `service_role secret key`

Save them somewhere private.

## 5.2 Configure Supabase Auth

Go to **Supabase → Authentication → URL Configuration**.

Set:

- `Site URL` = your production Vercel URL
- Add local dev URL if you use local testing

Use values like:

```text
Production site URL:
https://YOUR-PROJECT.vercel.app

Local development URL:
http://localhost:3000
```

If you later add a custom domain, update `Site URL` to that custom domain.

## 7. Quick SQL Verification Checks

After the migrations run, execute the following SQL in Supabase SQL Editor.

## 7.1 Check important tables exist

```sql
select table_name
from information_schema.tables
where table_schema = 'public'
  and table_name in (
    'organizations',
    'org_users',
    'sources',
    'rss_items',
    'ai_findings',
    'proposed_updates',
    'flightbooks',
    'flightbook_sections',
    'flightbook_exports',
    'training_programmes',
    'training_phases',
    'training_lessons',
    'lesson_documents',
    'document_assignments',
    'acknowledgements',
    'training_signoffs',
    'training_forms',
    'training_form_submissions',
    'organization_branding',
    'onboarding_checklists'
  )
order by table_name;
```

## 7.2 Check important views exist

```sql
select table_name
from information_schema.views
where table_schema = 'public'
  and table_name in (
    'v_dashboard_stats',
    'v_update_queue'
  )
order by table_name;
```

## 7.3 Check storage buckets exist

```sql
select id, name, public
from storage.buckets
order by id;
```

You should see buckets including:

- `flightbooks`
- `exports`
- `easa-source-files`

## 7.4 Check seeded sources

```sql
select type, active, count(*) as count
from sources
group by type, active
order by type, active desc;
```

## 7.5 Check `app_role` values

```sql
select enumlabel
from pg_enum
where enumtypid = 'app_role'::regtype
order by enumsortorder;
```

You should see role values including:

- `admin`
- `viewer`
- `instructor`
- `student`
- `compliance_manager`

## 8.3 CLI commands

If you already have the Supabase CLI installed, run:

```bash
supabase login
supabase link --project-ref YOUR_SUPABASE_PROJECT_REF
```

Then set Supabase function secrets:

```bash
supabase secrets set \
SUPABASE_URL=YOUR_SUPABASE_URL \
SUPABASE_SERVICE_ROLE_KEY=YOUR_SUPABASE_SERVICE_ROLE_KEY \
OPENAI_API_KEY=YOUR_OPENAI_API_KEY \

ANTHROPIC_API_KEY=YOUR_ANTHROPIC_API_KEY \
GOOGLE_API_KEY=YOUR_GOOGLE_API_KEY
```

You do not need every AI key.

Use only the ones you actually have.

If you are using only OpenAI, this shorter version is enough:

```bash
supabase secrets set \
SUPABASE_URL=YOUR_SUPABASE_URL \
SUPABASE_SERVICE_ROLE_KEY=YOUR_SUPABASE_SERVICE_ROLE_KEY \
OPENAI_API_KEY=YOUR_OPENAI_API_KEY \
```

Now deploy the functions:

```bash
supabase functions deploy rss-ingest
supabase functions deploy regulation-ingest
supabase functions deploy ai-analyze
supabase functions deploy apply-update
supabase functions deploy rollback
supabase functions deploy notifications
```

## 11. What To Configure Inside the App

## 11.1 Setup tab

Open `Settings → Setup`.

Check that:

- the organization exists
- your user is linked
- active RSS feeds exist
- AI config exists
- schedule exists

If feeds are missing, use:

```text
Restore EASA defaults
```

## 11.2 AI settings tab

Open `Settings → AI settings`.

For the easiest setup:

- provider: `openai`
- model: choose the model you want to use
- API key: your OpenAI API key

If you prefer Anthropic or Google, you can use those instead.

Important beginner note:

- using OpenAI is the easiest path if you also want embeddings
- if you do not configure embeddings, the app falls back to weaker keyword retrieval

## 11.3 Automation tab

Open `Settings → Automation`.

Recommended beginner values:

- Pipeline enabled: `Enabled`
- Runs per day: `1`
- First run time (UTC): choose a time you understand clearly
- Default export format: your preference
- Auto-approve low risk updates: `Manual review only` for now

Save the settings.

## 11.4 Branding tab

Open `Settings → Branding`.

Fill in:

- public school name
- primary color
- secondary color
- support email
- support phone

Save it.

This should affect the workspace branding in the app shell.

## 11.5 Onboarding tab

Open `Settings → Onboarding`.

Use it as your “go live” checklist.

It will auto-detect some steps from real app data and lets you manually mark the rest complete.

## 11.6 Exports tab

Open `Settings → Exports`.

This gives you CSV exports for:

- manual version register
- training assignments
- acknowledgements
- instructor sign-offs
- form submissions

## 18. Troubleshooting

## 18.1 Build succeeds locally but production app fails

Check:

- Vercel environment variables were added to the correct environment
- you redeployed after adding them
- `SUPABASE_SERVICE_ROLE_KEY` is present

## 18.2 Login works but app looks empty

Check:

- migrations were actually run
- `001_easa_sources.sql` and `002_html_sources.sql` were run
- your user is linked to an org
- `/settings?tab=setup` shows expected status

## 18.3 Pipeline fails

Check:

- `rss-ingest` deployed
- `regulation-ingest` deployed
- `ai-analyze` deployed
- Supabase function secrets were set
- AI settings were saved in the app

## 18.4 Search page shows results but no AI answer

That usually means:

- no AI provider is configured in app settings, or
- no AI env key exists in Vercel / Supabase secrets

## 18.5 Branding or onboarding does not save

That usually means the Phase 3 migrations were not applied yet:

- `021_training_school_ops.sql`
- `004_training_school_ops.sql`

## 18.6 Exports are empty

That usually means:

- you have not created any manual exports yet
- you have not created training assignments / acknowledgements / sign-offs yet
- `018_flightbook_exports.sql` was not applied
